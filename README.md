## 📖 About This Project

This repository is an **unofficial, non-commercial source reconstruction** of the original `creepyhorse.jar` Minecraft horror mod, originally authored by **PKFL**, as featured in the legendary video series by **LookOut3D**.

The sole purpose of this project is **reverse-engineering research and educational study** of the mod's underlying implementation. This is **not** a fork, successor, or replacement of the original mod.

> ⚠️ **Disclaimer:** All original art assets, gameplay concepts, and creative direction belong to their respective authors. This repository is strictly for technical research.

---

## ✨ What Was Done

The original `.jar` was decompiled using industry-standard tools (Fernflower / CFR). Due to the nature of Java bytecode decompilation, the raw output contained **hundreds of syntax and logic errors** stemming from:

- **Type Erasure** — Generic type information lost at the bytecode level, causing incorrect raw-type casts throughout the codebase.
- **Forced Casts** — Decompilers inserting incorrect or redundant `(Type)` casts that break compilation.
- **Control Flow Obfuscation** — Malformed `switch-case` structures, unreachable branches, and broken `try-catch` blocks generated by the decompiler's CFG reconstruction.

Every single one of these errors has been **manually identified, analyzed, and corrected** to produce a clean, compilable source tree.

Additionally, all **art assets and data files** originally embedded in the `.jar` (textures, models, sounds, `assets/`, `data/`) have been fully extracted and reintegrated into `src/main/resources/`, matching the standard Forge MDK project layout.

## 🚀 Building Locally

### Prerequisites

- **JDK 17** installed and set as your active Java runtime.
- **IntelliJ IDEA** (recommended) with the [Minecraft Development](https://plugins.jetbrains.com/plugin/8327-minecraft-development) plugin.
- A stable internet connection for Gradle to resolve Forge dependencies on first run.

### Step 1 — Clone the Repository

```bash
git clone https://github.com/Memorial1337/creepyhorse-reconstructed.git
cd creepyhorse-reconstructed
```

### Step 2 — Set Up the Workspace

Open the cloned folder in **IntelliJ IDEA** as a Gradle project. The IDE will automatically invoke Gradle to download Forge, deobfuscate the Minecraft jar, and configure the development workspace.

> It is strongly recommended to install the **Minecraft Development** plugin beforehand for full run configuration and mixin support.

### Step 3 — Build the Mod

```bash
./gradlew build
```

The compiled `.jar` will be output to `build/libs/`.

### Step 4 — Launch the Development Client

```bash
./gradlew runClient
```

This launches a development Minecraft instance with the mod hot-mounted for in-game testing and debugging.

## ✨ 重构内容

原始 `.jar` 使用业界标准工具（Fernflower / CFR）进行反编译。由于 Java 字节码反编译的固有局限性，原始输出包含**数百处语法与逻辑错误**，主要来源于以下三类问题：

- **类型擦除（Type Erasure）** — 泛型类型信息在字节码层面丢失，导致全局范围内出现大量错误的原始类型强转。
- **强制类型转换（Forced Casts）** — 反编译器自动插入了大量错误或多余的 `(Type)` 强转表达式，破坏正常编译。
- **控制流混淆（Control Flow Obfuscation）** — 由反编译器控制流图重建过程产生的格式错误的 `switch-case` 结构、不可达分支及损坏的 `try-catch` 块。

上述所有错误均已被**手动逐一识别、分析并修正**，最终产出一套干净、可正常编译的完整源码树。

此外，原 `.jar` 中内嵌的所有**美术资产与数据文件**（贴图、模型、音效、`assets/`、`data/` 目录）均已被完整提取，并按标准 Forge MDK 项目结构重新整合至 `src/main/resources/` 目录下。

## 🚀 本地构建步骤

### 前置条件

- 已安装 **JDK 17** 并将其设置为当前活跃 Java 运行时。
- 已安装 **IntelliJ IDEA**（推荐），并安装 [Minecraft Development](https://plugins.jetbrains.com/plugin/8327-minecraft-development) 插件。
- 首次运行时需要稳定网络连接，以供 Gradle 解析 Forge 依赖。

### 第 1 步 — 克隆仓库

```bash
git clone https://github.com/Memorial1337/creepyhorse-reconstructed.git
cd creepyhorse-reconstructed
```

### 第 2 步 — 配置开发工作区

用 **IntelliJ IDEA** 以 Gradle 项目形式打开克隆目录。IDE 将自动调用 Gradle 下载 Forge 依赖、对 Minecraft jar 进行去混淆处理，并完成开发工作区的全部配置。

> 强烈建议提前安装 **Minecraft Development** 插件，以获得完整的运行配置与 Mixin 支持。

### 第 3 步 — 构建模组

```bash
./gradlew build
```

编译完成后，输出的 `.jar` 包位于 `build/libs/` 目录下。

### 第 4 步 — 启动开发客户端

```bash
./gradlew runClient
```

此命令将以模组热挂载状态启动一个 Minecraft 开发客户端，供游戏内实时测试与调试使用。
